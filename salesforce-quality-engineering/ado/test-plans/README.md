# Test Plans

## Purpose

Container for suites and cases used to evidence a sprint/release/UAT cycle.

## When Used

Per sprint, release, SIT, UAT, or production validation cycle.

## Inputs

- Test Plan intent
- Scope
- Area/Iteration
- Suite strategy

## Outputs

- Test Plan
- Suite structure
- Execution evidence

## Best Practices

- One clear purpose per plan
- Align naming to release/sprint
- See suite organization guide

## Available Documents

| Document | Focus |
|----------|-------|
| [Test Plan](test-plan.md) | Test Plan work item / artifact |

## Scope note

**Sprint 6 focuses on Delivery Intelligence** — work item quality, relationships, and ADO-ready artifact content.  
**Not in scope:** Azure DevOps REST API automation / auto-publish (optional later enhancement).
