# Tasks

## Purpose

Implementation/test activities that complete a User Story.

## When Used

Sprint planning/breakdown for build, config, or test execution work.

## Inputs

- User Story
- Team capacity

## Outputs

- Task work items with remaining work

## Best Practices

- Small, completable in a sprint
- Separate test tasks from build tasks when useful
- Never hide a story inside a task
