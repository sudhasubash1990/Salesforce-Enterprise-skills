# Dashboards

## Purpose

Role-based dashboard designs with KPIs and decision support.

## When Used

Team, release, and executive governance cadences.

## Inputs

- Audience
- Decisions to support
- Available queries/widgets

## Outputs

- Dashboard design specs (not widget JSON)

## Best Practices

- One primary decision per dashboard
- Show residual risk, not vanity metrics
- Align filters to Area/Iteration
