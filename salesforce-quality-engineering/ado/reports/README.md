# Reports

## Purpose

ADO-oriented reporting packages that complement Sprint 5 report templates.

## When Used

Sprint end, release gates, executive reviews.

## Inputs

- Execution data
- Defects
- RTM/coverage

## Outputs

- Report outlines mapped to ADO queries/dashboards

## Best Practices

- Reuse Sprint 5 report templates
- Cite query sources
- No invented metrics

| Document | Focus |

## Scope note

**Sprint 6 focuses on Delivery Intelligence** — work item quality, relationships, and ADO-ready artifact content.  
**Not in scope:** Azure DevOps REST API automation / auto-publish (optional later enhancement).
