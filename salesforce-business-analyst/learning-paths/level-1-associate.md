## Practice Exercises

1. **User story pack** — Write 5 user stories with AC for a simple lead-to-opportunity flow using the story template
2. **Meeting minutes** — Document a mock discovery session using the meeting minutes template
3. **Stakeholder map** — Create a RACI-style stakeholder matrix for a fictional CRM rollout
4. **Requirement classification** — Categorize 10 sample statements as business, functional, or NFR

## Exit Checklist

- [ ] Can explain difference between business and functional requirements
- [ ] Produces INVEST-compliant user stories with testable AC
- [ ] Documents workshop outputs without losing traceability to source
- [ ] Identifies key stakeholders and their interests
- [ ] Uses repository templates without structural errors
- [ ] Passes L1 interview topics: [interview-guide/business-analysis.md](../interview-guide/business-analysis.md)
