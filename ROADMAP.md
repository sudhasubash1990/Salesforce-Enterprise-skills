# Roadmap

Strategic evolution plan for Salesforce Enterprise Skills.

## Vision

Become the authoritative, agent-ready knowledge base for Salesforce consulting disciplines—starting with Business Analysis and expanding to Solution Architecture, Project Management, Change Management, and Testing.

## Phase 1 — Foundation (Current)

- [x] Repository governance and quality framework
- [x] Shared glossary, taxonomy, and capability map
- [x] Salesforce Business Analyst skill (v0.1)
- [x] Sprint 1 — Business Analyst Brain (v0.2)
- [x] Sprint 2 — Enterprise Knowledge Base (v0.3)
- [x] Sprint 3 — Enterprise Template Library (v0.4)
- [x] Sprint 4 — Enterprise Playbook Library (v0.5)
- [x] Sprint 5 — Industry Scenario Library (v0.6)
- [x] Sprint 6 — Enterprise Interview Intelligence Framework (v1.0)
- [x] Sprint 7 — Cross-Linking & Metadata Framework (v1.1)
- [x] Sprint 8 — Enterprise Validation (v1.2)
- [x] Sprint 9 — Continuous Knowledge Management (v1.3)
- [x] Sample artifacts and industry scenarios
- [ ] Community review and pilot on 2–3 live programs

## Phase 2 — Depth (Q3 2026)

- [ ] Expand knowledge base: CPQ, Revenue Cloud, Service Cloud Voice (Sprint 2 core complete)
- [ ] Add integration and data migration playbooks
- [ ] Release formal user story quality rubric with automated lint rules
- [ ] Add `.cursor/skills/` symlink for native Cursor skill discovery

## Phase 3 — Adjacent Disciplines (Q4 2026)

- [ ] Salesforce Solution Architect skill
- [ ] Salesforce Project Manager / Scrum Master skill
- [ ] Salesforce QA / Test Lead skill
- [ ] Cross-skill traceability matrix (requirement → design → test)

## Phase 4 — Enterprise Scale (2027)

- [ ] Industry accelerators (FS, HLS, CMT, Public Sector packs)
- [ ] Metadata-driven skill configuration per client engagement
- [ ] Integration with ADO/Jira/Confluence export templates
- [ ] Certification alignment guides (Application Architect, Business Analyst)

## Guiding Principles for Roadmap Decisions

1. **Outcome over output** — Skills must improve delivery outcomes, not just document volume.
2. **Agent-first, human-validated** — Content must be machine-actionable and human-reviewable.
3. **Progressive disclosure** — Keep skill entry points concise; depth lives in linked files.
4. **Traceability** — Every requirement pattern must connect to design and test patterns.
